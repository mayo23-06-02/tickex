## üìù Documentation Standards

### File Naming
- Use kebab-case: `frontend-tickets-page.md`
- Prefix with module: `backend-`, `frontend-`
- Be descriptive: `component-index.md` not `components.md`

### Structure
- Start with metadata (date, status, module)
- Use numbered sections
- Include table of contents for long docs
- Add code examples
- Include "Last Updated" date

### Code Examples
- Use TypeScript
- Show complete examples
- Include imports
- Add comments for clarity

### Formatting
- Use Markdown
- Use code blocks with language
- Use tables for structured data
- Use lists for items
- Use headings for hierarchy
